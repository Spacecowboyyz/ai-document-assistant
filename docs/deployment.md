# Production deployment guide

Deploy the AI Document Assistant using **Render** (backend) and **Vercel** (frontend). Both offer free tiers suitable for demos and personal projects.

---

## Architecture

```
Browser → Vercel (Next.js)
            ↓ HTTPS
         Render (FastAPI Docker + persistent disk at /app/data)
            ↓
         Groq API (chat) + sentence-transformers (embeddings in container)
```

| Component | Host | Notes |
|-----------|------|--------|
| Frontend | Vercel | `NEXT_PUBLIC_API_URL` points to Render |
| Backend | Render | `render.yaml` at repo root |
| SQLite, PDFs, Chroma | Render disk | Mounted at `/app/data` |
| Chat | Groq | Free API key required |

Local development uses **Ollama** (`AI_PROVIDER=ollama`). Production uses **Groq** (`AI_PROVIDER=groq`).

---

## Get a free Groq API key

1. Open [https://console.groq.com](https://console.groq.com) and sign up (free).
2. Go to **API Keys** → **Create API Key**.
3. Copy the key (`gsk_...`). You will paste it into Render — **never commit it to git**.

Default chat model in the app: `llama3-8b-8192` (set `GROQ_CHAT_MODEL` on Render if you want to override).

## Step 1 — Deploy backend on Render

### 1.1 Push code to GitHub

Render deploys from your Git repository. Commit and push `main` (or your default branch).

### 1.2 Create a Blueprint from `render.yaml`

1. Go to [https://dashboard.render.com](https://dashboard.render.com) and sign in (GitHub is easiest).
2. Click **New +** → **Blueprint**.
3. Connect the repository that contains this project.
4. Render reads **`render.yaml`** at the repo root and proposes the `ai-doc-assistant-backend` web service with a **1 GB persistent disk** at `/app/data`.

Review the plan, then click **Apply**.

### 1.3 Set secrets in the Render dashboard

`render.yaml` marks these as manual (not stored in git):

| Variable | What to set |
|----------|-------------|
| `GROQ_API_KEY` | Your `gsk_...` key from Groq |
| `CORS_ORIGINS` | Your Vercel URL after Step 2, e.g. `https://your-app.vercel.app` (no trailing slash) |

Other variables are defined in `render.yaml`:

| Variable | Value |
|----------|--------|
| `AI_PROVIDER` | `groq` |
| `SECRET_KEY` | Auto-generated by Render |
| `DATABASE_URL` | `sqlite:////app/data/app.db` |
| `CHROMA_DB_PATH` | `/app/data/chroma_db` |
| `UPLOADS_DIR` | `/app/data/uploads` |

Optional extras in the dashboard:

| Variable | Example |
|----------|---------|
| `GROQ_CHAT_MODEL` | `llama3-8b-8192` |
| `ENVIRONMENT` | `production` |

Render sets **`PORT`** automatically; the Docker image listens on that port.

### 1.4 Wait for deploy and copy your backend URL

1. Open the service → **Logs** and wait until the deploy is **Live**.
2. Copy the public URL, e.g. `https://ai-doc-assistant-backend.onrender.com`.
3. **Health check:** open `https://YOUR-RENDER-URL/health` — expect JSON with `"status":"ok"`.
4. **Models:** open `https://YOUR-RENDER-URL/api/v1/models/status` — expect `"ai_provider":"groq"` and `"models_ready":true` when `GROQ_API_KEY` is set.

**Free tier note:** Render may spin down idle services; the first request after sleep can take 30–60 seconds.

## Step 2 — Deploy frontend on Vercel

### 2.1 Import the repository

1. Go to [https://vercel.com](https://vercel.com) → **Add New** → **Project**.
2. Import the same GitHub repository.
3. Set **Root Directory** to `frontend`.
4. Framework: **Next.js** (auto-detected).

### 2.2 Environment variable

| Name | Value |
|------|--------|
| `NEXT_PUBLIC_API_URL` | `https://YOUR-RENDER-URL` (no trailing slash) |

Use the exact Render URL from Step 1.4. This is used for PDF upload and chat streaming (SSE).

### 2.3 Deploy

Click **Deploy**. When finished, copy your Vercel URL (e.g. `https://your-app.vercel.app`).

---

## Step 3 — Connect backend and frontend

1. In **Render** → your backend service → **Environment**:
   - Set `CORS_ORIGINS` to your **Vercel URL** (exact match, no trailing slash).
2. Click **Save Changes** (Render redeploys automatically).
3. In **Vercel** → **Settings** → **Environment Variables**:
   - Confirm `NEXT_PUBLIC_API_URL` matches your Render URL.
4. If you change either URL later, update both sides and redeploy.

**Flow:**

- Browser loads the app from Vercel.
- Auth JSON calls can use Next.js rewrites in production when `NEXT_PUBLIC_API_URL` is set.
- Upload and chat call the Render backend directly via `NEXT_PUBLIC_API_URL`.

---

## Step 4 — Smoke test

1. Open your Vercel URL.
2. **Register** a new account.
3. **Upload** a PDF and wait for indexing.
4. Open the document and **chat** — you should see streaming text and sources.

```bash
# backend/.env
AI_PROVIDER=groq
GROQ_API_KEY=gsk_your_key

# From repo root
docker compose -f docker-compose.prod.yml up --build
```

Frontend:

```bash
cd frontend
echo "NEXT_PUBLIC_API_URL=http://localhost:8000" > .env.local
npm run dev
```

## ChromaDB and provider switching

Vectors are stored under **`{CHROMA_DB_PATH}/{AI_PROVIDER}/`** (e.g. `.../chroma_db/groq`). Do not mix Ollama (768-dim) and Groq/MiniLM (384-dim) data in the same provider folder.

## Troubleshooting

| Problem | Likely cause | Fix |
|---------|--------------|-----|
| CORS error | Wrong `CORS_ORIGINS` | Exact Vercel URL on Render, redeploy |
| `models_ready: false` | Missing `GROQ_API_KEY` | Set in Render environment |
| Slow first request | Render free spin-down | Wait and retry |
| Chat 429 | Groq rate limit | Pause, retry |
| Upload fails on Vercel | Wrong API URL | Check `NEXT_PUBLIC_API_URL` |
| Data lost after deploy | Disk not attached | Confirm `render.yaml` disk block applied |
| Build fails / OOM | Large ML image | Retry deploy; first embed loads the model |
| Session expired | JWT TTL | Log in again |
